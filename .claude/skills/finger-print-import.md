# Fingerprint Attendance Import

HR uploads the two fingerprint machines' monthly exports; punches land in `time_attendences`,
then the existing Punch Card → PaySlip flow takes over (see [[punch-card-payroll]]). Scope is
**import punches only** — it does NOT generate salary notes/payslips (HR still allocates worked-days
into Daily/Sunday/Holiday in the Punch Card). Route `/import_attend` →
`FullCircleWeb.UploadPunchLog.Index` (`lib/full_circle_web/live/time_attend_live/upload_punch_log.ex`).

## Pipeline

`.xls` (browser: SheetJS → `.xlsx`) → `read_excel_files/1` (xlsx_reader, "Att.log report" sheet) →
`FingerPrintImport.parse_files_rows/1` → `fill_in_employee_info/3` (match by `punch_card_id`) →
`build_people/3` (matcher panel) → `confirm_match`/`manual_match` persist the key →
`insert_time_attendence_from_logs/3` (idempotent).

## `.xls` is read in the BROWSER, not the server

The fingerprint machines emit legacy `.xls` (BIFF/OLE2); `xlsx_reader` **cannot** read it (it errors
`"invalid zip file"`). `Hooks.XlsToXlsxUpload` (`assets/js/app.js`) converts `.xls`/`.xlsx`→`.xlsx`
in-browser via vendored SheetJS (`assets/vendor/sheetjs/xlsx.mjs`, dynamically imported so it's a lazy
chunk, not in the main bundle), then `this.upload("xlsx_file", convertedFiles)`. The server's
`allow_upload(:xlsx_file, accept: ~w(.xlsx))` therefore only ever sees `.xlsx` — **don't change the
server read path to handle `.xls`**. The input is a plain `<input phx-hook="XlsToXlsxUpload"
phx-update="ignore">`, NOT `live_file_input`.

## `FullCircle.HR.FingerPrintImport` (pure, no IO/DB)

Public: `parse_file_rows/1` (one file's rows → punch maps), `parse_files_rows/1` (list of files),
`file_date_range/1` (`{from, to}`). `fill_flags_to_map/1` is private.

Each punch map: `%{stamp: Time, flag:, ID:, Name:, Department:, punch_card_id:, punch_time_local:}`.

### "Att.log report" sheet layout (0-indexed rows)
- row 0 title, row 1 blank, **row 2 idx 2** = `"2026-01-01 ~ 2026-01-31"`, row 3 day-number header,
  **rows 4+ = `[info_row, attendance_row]` pairs** per employee.
- Attendance cell = a day's punches concatenated, e.g. `"07:5512:0112:5417:01"`; the parser regex-scans
  `\d{2}:\d{2}`, **dedups punches within 10 min**, assigns positional flags
  `1_IN_1,1_OUT_1,2_IN_2,2_OUT_2,3_IN_3,3_OUT_3` then `nil` for a 7th+ punch.

### Info-row index gotcha
`filter_times_n_split_to_map` does `Enum.reject(info, & &1 == "")` FIRST, then reads `Enum.at(info, 1)`
= ID, `(info, 3)` = Name, `(info, 5)` = Department. A real info row is
`["ID:","",id,"","","","","","Name:","",name,...,"Dept:",dept]`; after rejecting blanks it compresses
to `["ID:", id, "Name:", name, "Dept:", dept]` so 1/3/5 line up. **In test fixtures you MUST include
the `"Dept:"` label before the dept value**, or `(info, 5)` is `nil`.

## `punch_card_id` is canonical `"<id>.<name>"` (department excluded)

E.g. `"4.Linda Santika"`. Built as `"#{Enum.at(info,1)}.#{Enum.at(info,3)}"` (no space-stripping).
Why this exact shape:
- The fingerprint **ID alone collides across the two machines** — the same number is a *different*
  person on each (machine 1 ID 5 = "Chaudhary Ramesh Kumar", machine 'a' ID 5 = "P.Sook Kee").
  id+name disambiguates; `"11.Gurung Chantala"` and `"11.Marianus Wasa Pae"` coexist as distinct keys.
- A person's **department can differ between machines/months**, so including dept would make the same
  person's key unstable — hence dept is dropped.

This string is generated by the importer AND stored on `employees.punch_card_id`. Changing the format
orphans previously-matched employees (one-time re-match via the panel). Older data may carry legacy
`"Name.ID.Dept"` or `"ID.Name.Dept"` strings — re-match rather than migrate (names contain dots).

## Multi-file union (`parse_files_rows/1`)

The two machines cover **disjoint** employee populations, so it just `flat_map`s each file's
`parse_file_rows/1` — no per-employee merge. Returns `{:error, {:date_range_mismatch, ranges}}` if the
files don't all cover the same month (guards against mixing months). `read_excel_files/1` returns
`{:ok, rows} | {:error, _}` (via `with`) so a non-machine `.xlsx` flashes an error instead of crashing.

## 3-tier matcher

`build_people/3` groups attendances by `punch_card_id` (people with **zero punches never appear** —
attendances only contains parsed punches) and per person:
1. **Exact** — `fill_in_employee_info/3` matched it (`employee_id != "!! Not Found !!"`) → green row.
   Uses `HR.get_employees_by_punch_card_ids/3`, which has **no status filter**, so an already-matched
   resigned employee with final-month punches still resolves.
2. **Fuzzy** — `HR.match_employee_candidates(name, company, user, 3)` → top-3 buttons. It ranks ALL
   statuses (Active-first via a `CASE WHEN status = 'Active'` order, then `similarity_order([:name],..)`)
   so resigned staff are matchable. Do NOT use `HR.employees/3` here — it's Active-only.
3. **Manual** — a `<select>` over `@all_employees` (= `match_employee_candidates("", …, 10_000)`).

Confirm/pick calls `HR.set_employee_punch_card_id/4` (persists for future auto-match), then re-resolves.
Render events: `confirm_match` params are kebab `%{"card-id","emp-id"}` (from `phx-value-card-id`);
`manual_match` params are `%{"card_id","employee_id"}` (form field names). Keep them distinct.

## `StdInterface.update` audit-log `_id`-key gotcha

`set_employee_punch_card_id/4` must pass a non-`_id` field alongside `punch_card_id`:
`%{"punch_card_id" => key, "name" => emp.name}`. `Sys.attr_to_string/1` (audit-log delta builder)
**drops any key ending in `_id`** (plus `delete/__meta__/photo_data/photo_descriptor`); updating only
`punch_card_id` yields an empty `delta`, and `Sys.Log`'s `validate_required(:delta)` then fails the
whole `Multi`. Including `"name"` (unchanged) gives a non-blank delta.

## Idempotent insert

`HR.insert_time_attendence_from_log/2` skips inserting when a same-`flag` punch already exists within
±5 min for that employee/company, so re-imports and overlapping data are safe. The plural wrapper also
skips unmatched people and `nil`-flag (7th+) punches.

## Verifying against real files
Convert `.xls`→`.xlsx` with the vendored SheetJS in Node, then feed through `read_excel_files/1` +
`parse_files_rows/1` and compare per-person distinct-punch-days to the machine's "Att. Stat."
real-days. Design/plan: `docs/superpowers/specs/2026-06-08-fingerprint-xls-import-design.md`,
`docs/superpowers/plans/2026-06-08-fingerprint-xls-import.md`.
